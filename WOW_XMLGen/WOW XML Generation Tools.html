Tool to Generate Alert and Work Order XML Files for Ingestion

This is a Perl script that will work on Windows or Unix and uses only standard Perl modules.

Download here.

Usage: [perl] genXmlForWOWIngestion.pl [-h] | [-x (alert|wo|so) -n number_of_records

              -s start_record -t tenant -d days_in_past -i increment_in_seconds -u]
Required parameters:
Optional parameters:
    -x: type of xml to output, alert=alerts, wo=work orders, so=service orders (default 'alert')
    -n: number of records (default 100)
    -s: start record number or start record ID (default 1)
    -t: Tenant (default VDS_LILLE)
    -d: Days in past from which to start generating timestamps (default 25)
    -i: Number of seconds to increment timestamps for each alert (default 180)
    -u: Update instead of insert
    -o: String to append to the base output file name

 ----------------------------------------------------------------------------------------------------------------------------------
Example 1: ./genXmlForWOWIngestion.pl -x wo -n 10000 -s 100 -t VDS_LYON
        Generate 10,000 Workorder XMLs for tenant VDS_LYON, starting with ID 100

Tool to Generate Reading XML Files for Ingestion

This is a Perl script that will work on Windows or Unix and uses only standard Perl modules.

Download here.

Usage: [perl] genReadingsForWOWIngestion.pl [-h] | -f input_file [-n iterations -d days_in_past -i increment_in_seconds -t tenant]
Required parameters:
    -f: Input file, with format as follows:
        Asset Type ID, External Asset ID, Measurement Type ID, Range Start, Range End, Integer(0|1)
        Examples:
        Esri_wHydrant,607456,ENABLED,0,2,1
        Esri_wMain,607456,PRESSURE,-10.5,50,0

        The script will generate a reading value between Range Start and Range End, converting to Integer if the last value is 1
Optional parameters:
    -n: Number of iterations to generate readings for the input file (default 1)
    -d: Number of Days in Past. Generated reading time stamps will start from this point (default 10)
    -i: Increment in seconds. The timestamp for each block of readings is incremented by this amount (default 900)
    -t: Tenant (default VDS_LILLE)
 ----------------------------------------------------------------------------------------------------------------------------------
Example 1: ./genReadingsForWOWIngestion.pl -f Hydrants1.txt -n 5 -d 50 -i 3600 -t VDS_LYON
        Load template for readings from Hydrants1.txt, generate 5 times the number of entries in Hydrants1.txt
            starting with a timestamp 50 days ago and incrementing 1 hour for each block of entries in Hydrants1.txt

 

You must pass in a template file with the following information:

Asset Type ID, External Asset ID, Measurement Type ID, Range Start, Range End, Integer(0|1)

Tool to Generate Asset XML Files for Ingestion

This is a Java program that should work equally well on Linux and Windows.

Download here.

Command arguments : <FLOWTYPE> <USECASE> <TENANT> <GENERATED DATA SIZE> <INI> [<FILESUFFIX>]
[FLOWTYPE]: ASSET|READINGS|WO|SO|ALERTS
[USECASE]: INSERT/UPDATE/DELETE
[TENANT]:tenant ID (e.g. VDS_LILLE).
[GENERATED DATA SIZE]: e.g. 100
[INI]:Initialisation file
[FILESUFFIX]:Optional suffix to base file name (default 1)

Example:

java -jar AssetXmlGenerator.jar asset insert VDS_LILLE 100 Esri.wHydrant.ini 1

 

The key parameter is an initialization file defining the content of the generated assets.

A couple of examples are shown below (grammar for each section is described by comments)

Asset type Esri.wHydrant

[ASSETTYPE]

Type=Esri.wHydrant
; For ExternalID and Label you can specify %s to generate sequential number, starting with the value after the semi-colon (or 1 default)
; For ExternalID and Label you can specify %r to generate random number between 10,000 and 100,000
ExternalID=H%s;1
Label=Hydrant %s;1
Description=Hydrant
; Base Latitude;Delta Latitude;Base Longitude;Delta Longitude (default location around Lille)
Location=50.6033;0.06;2.9655;0.18


[MEASUREMENTS]

; MeasurementName=Unit(can be blank, but leave semi-colon);acceptable min/max;caution min/max;critical min/max

ENABLED=;0:-0.5/0.5;1:0.5/1.5;2:1.5/2.5

[PROPERTIES]

; Property definition types
; PropertyName=date;date format;initial year (will generate random date between jan 1 of this year and current date)
; PropertyName=string;range:start_of_range,end_of_range (generate random number between start and end of range)
; PropertyName=string;enum:value1,value2, ...  (select random value from list of values)
; PropertyName=double;start_of_range,end_of_range;unit (generate random value between start and end of range, unit is optional)
; PropertyName=integer;start_of_range,end_of_range;unit (generate random integer between start and end of range, unit is optional)

INSTALLDATE=date;yyyy-MM-dd HH:mm:ss;2000
MANUFACTURER=string;enum:Manufacturer1,Manufacturer2,Manufacturer3
ENABLED=string;range:0,1
ACTIVEFLAG=string;enum:True,False
OWNEDBY=string;enum:Owner1,Owner2,Owner3
MAINTBY=string;enum:Maintainer1,Maintainer2,Maintainer3
DIAMETER=double;0,100;mm
TYPE=string;enum:Hydrant
FACILITYID=string;range:10000,100000
